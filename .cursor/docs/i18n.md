# Internacionalização (PT + EN)

## Decisão

| Item | Escolha |
|------|---------|
| Idiomas | **PT-BR** (principal) + **EN** |
| Tradução automática | **DeepL API Free** |
| Armazenamento | Campos bilíngues no MySQL (JSON ou `spatie/laravel-translatable`) |
| Tradução em tempo real no site | **Não** — EN gerado no admin e salvo no banco |

---

## UX do painel admin

Cada **campo traduzível** exibe **tabs PT-BR | EN**.

```
┌─────────────────────────────────────────────┐
│  Título do projeto                          │
│  [ PT-BR ]  [ EN ]          [ Gerar EN ↻ ]  │  ← botão geral por formulário
├─────────────────────────────────────────────┤
│  (conteúdo da tab ativa)                    │
└─────────────────────────────────────────────┘
```

### Comportamento

| Ação | Resulto |
|------|---------|
| Tab **PT-BR** | Edição do conteúdo fonte |
| Tab **EN** | Visualização/edição do inglês |
| **Gerar EN** (botão geral) | DeepL traduz **todos os campos traduzíveis** do formulário a partir do PT-BR |
| Editar campo EN | Sobrescreve tradução automática — valor manual tem prioridade |
| Salvar | Persiste `{ pt, en }` por campo no MySQL |

### Regras

- PT-BR é sempre a **fonte da verdade** para geração automática
- EN gerado é **editável** — correções manuais não são sobrescritas até novo clique em **Gerar EN**
- **Gerar EN** deve avisar se EN já foi editado manualmente (confirmar sobrescrita)
- Campos **não traduzíveis**: slug, URLs, stack tags, números de métricas, nomes de empresas

## DeepL API Free

| Item | Valor |
|------|-------|
| Plano | Free |
| Limite | ~500.000 caracteres/mês |
| Par | PT → EN |
| Config | `DEEPL_API_KEY` no `.env` do Laravel |

### Fluxo Laravel

```
POST /admin/translate
  → lê campos PT do formulário
  → chama DeepL API (batch por campo ou um request)
  → retorna EN para preencher tabs (Inertia/React)
  → usuário revisa → salva no banco
```

## Next.js (site público)

- **`next-intl`** — UI estática em `messages/pt.json` e `messages/en.json`
- **Toggle PT | EN** no header — redireciona para a rota equivalente no outro idioma
- **Prefixo na URL** por idioma
- Conteúdo dinâmico: API com `locale` derivado do prefixo

### Rotas por idioma

| Idioma | Prefixo | Exemplo |
|--------|---------|---------|
| PT-BR | `/pt` | `/pt/projetos` |
| EN | `/en` | `/en/projetos` |

### Raiz `/`

**Redireciona para `/pt` ou `/en` conforme cookie `locale`.**

| Situação | Destino |
|----------|---------|
| Primeira visita (sem cookie) | `/pt` (padrão) |
| Cookie `locale=pt` | `/pt` |
| Cookie `locale=en` | `/en` |

Middleware Next.js: `GET /` → lê cookie → `redirect(/pt)` ou `redirect(/en)`.

### Comportamento do toggle

1. Visitante em `/pt/projetos/niara` clica **EN**
2. Redireciona para `/en/projetos/niara`
3. Cookie `locale` persiste preferência (opcional, para próxima visita em `/`)
4. `next-intl` resolve mensagens UI + API recebe `Accept-Language: en`

### Mapeamento de rotas

| PT-BR | EN |
|-------|-----|
| `/pt` | `/en` |
| `/pt/projetos` | `/en/projetos` |
| `/pt/projetos/[slug]` | `/en/projetos/[slug]` |
| `/pt/sobre` | `/en/sobre` |
| `/pt/contato` | `/en/contato` |

> Slug do projeto permanece único (não traduzir URL slug).

## Campos traduzíveis por módulo

| Módulo | Traduzir | Não traduzir |
|--------|----------|--------------|
| Projetos | título, descrição, destaques, labels de métricas | slug, url, stack, valores numéricos |
| Hero/Sobre | headline, bio, CTAs (label) | nome, foto URL |
| Experiências | cargo, descrição, responsabilidades | empresa, período |
| Skills | — (nomes técnicos iguais) | nome da skill |
| SEO | title, description, og_* | rota, canonical |
| Contato | labels do formulário (UI) | URLs, e-mail |
